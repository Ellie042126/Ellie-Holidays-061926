Executive Summary The MedReg-AI Reviewer is an agentic artificial intelligence system designed to streamline the medical device regulatory clearance process. Specifically, it targets the transition from a 510(k) Summary (submitted by a manufacturer) to a 510(k) Review Report (authored by a regulatory reviewer). The system leverages a "Human-in-the-Loop" (HITL) architecture, utilizing Google Gemini 2.0 Flash (as a proxy for the requested next-gen Gemini models) to perform high-fidelity entity extraction and report synthesis. System Architecture & Agentic Workflow 2.1 Agentic Framework The system operates using a multi-agent orchestration pattern where specific "skills" are assigned to different logical instances of the LLM. The Extraction Agent (The "Prefiller"): Role: Parses unstructured text from a 510(k) summary. Objective: Map text segments to a structured 100-entity JSON schema. Logic: Employs Chain-of-Thought (CoT) reasoning to infer missing regulatory classifications based on context (e.g., inferring a Class II status from the product code LLZ). The Synthesis Agent (The "Writer"): Role: Transforms structured JSON data and raw summary text into a formal, narrative-based 510(k) Review Report. Objective: Maintain regulatory tone, ensure consistency between the "Subject Device" and "Predicate Device" comparisons, and incorporate specific reviewer comments. The Validation Agent (The "Auditor"): Role: Runs in the background during UI editing. Objective: Checks for logical inconsistencies (e.g., if the user marks the device as "Substantially Equivalent" but lists "Critical Deficiencies"). 2.2 Data Flow Pipeline Ingestion: User inputs 510(k) Summary and JSON Schema. Extraction: Prefiller Agent populates the 100-entity JSON. Refinement: Reviewer interacts with a Streamlit UI to edit the JSON values. Template Binding: Reviewer chooses a default or custom Markdown/Word report template. Generation: Synthesis Agent generates the draft Review Report. Final Edit: Reviewer performs final narrative edits and exports the document. 3. Download Button: Export the edited JSON for local archival. 3.3 Report Generation Module (Stage 3) Template Logic: The system supports dynamic placeholders (e.g., {{device_name}}). Contextual Memory: The Synthesis Agent receives the original Summary, the validated JSON, and any "Reviewer Questions" generated during the analysis. Markdown Preview: Real-time rendering of the generated report using st.markdown. 3.4 Export & Versioning Formats: .docx (via python-docx) for official submission and .md for quick sharing.
